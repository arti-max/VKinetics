# Шаг 4: Генерация Предмета (Baking Pipeline)

Поздравляю, ваш механизм работает в мире! Но если вы попытаетесь взять его в инвентарь, возникнет проблема: VoxelCore не умеет рендерить модели с костями (Rigged Models) в виде иконок инвентаря.

Для решения этой проблемы мы используем автоматический Python-скрипт `create_models.py`, который "запекает" (bakes) все трансформации костей в статичные примитивы и генерирует предметы-заглушки.

## Как это работает?

Вам не нужно писать JSON для предмета руками. Наш пайплайн сделает всё сам:

1. Он берет ваш файл `models/mechanical_fan.vcm`.
2. Схлопывает кость `rotor` и пересчитывает координаты всех её частей (запекает их).
3. Сохраняет статичную модель в `models/squash/mechanical_fan.vcm`.
4. Генерирует скрытый визуальный блок `blocks/itemm_mechanical_fan.json`, который использует "сплющенную" модель.
5. Генерирует предмет `items/mechanical_fan.item.json`, который ссылается на этот визуальный блок как на иконку (`icon-type: block`), а при клике (ПКМ) ставит настоящий блок из сети.

## Использование скрипта

1. Убедитесь, что ваша исходная модель лежит в `models/mechanical_fan.vcm`.
2. Убедитесь, что исходный JSON блока лежит в `blocks/mechanical_fan.json`.
3. Откройте терминал в корневой папке вашего контент-пака.
4. Запустите скрипт:

```bash
   python create_models.py
```

**Ожидаемый вывод в консоли:**

```text
--- Начало обработки ---
Модели: запечено и обработано файлов .vcm -> X
Блоки/Предметы: сгенерировано пар файлов -> X
--- Готово! ---

```

Всё! Запускайте игру, открывайте инвентарь - ваш механический вентилятор появится там с красивой объемной иконкой. При установке он автоматически подключится к кинетической сети и начнет вращаться, если к ней подведен генератор.
