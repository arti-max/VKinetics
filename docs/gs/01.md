# Шаг 1: Концепт и JSON Блока

В этом руководстве мы создадим **Механический вентилятор (Mechanical Fan)**. Это блок-потребитель, который подключается к кинетической сети, потребляет энергию (SU - Stress Capacity) и визуально вращает свои лопасти с нужной скоростью.

Так как в VoxelCore блоки по умолчанию статичны, мы используем паттерн **Block + ECS Entity**:

1. Физический невидимый блок хранит данные (в полях чанка) и обрабатывает коллизии.
2. Сущность (Entity) со скелетом отвечает за красивое, плавное вращение лопастей в каждом кадре.

## Создание JSON Блока

Создайте файл `blocks/mechanical_fan.json`.

Обратите внимание: мы указываем `"model": "none"`, чтобы скрыть стандартный рендер блока. Главная магия происходит в блоке `"fields"` - именно эти данные VoxelCore будет сохранять вместе с чанком, чтобы сеть не разваливалась при перезаходе в игру.

```json
{
    "model": "none",
    "hitbox": [0.0, 0.0, 0.0, 1.0, 1.0, 1.0],
    "rotation": "pipe",
    "light-passing": true,
    "sky-light-passing": true,
    "solid": false,
    "obstacle": true,
    "selectable": true,
    "texture": "stone",
    "fields": {
        "entity_uid": {"type": "int32", "convert-strategy": "reset"},
        "network_id": {"type": "int32"},
        "direction": {"type": "int8"},
        "ratio": {"type": "float32"},
        "roll": {"type": "int16", "convert-strategy": "reset"}
    }
}

```

* **`entity_uid`**: ID визуальной сущности, чтобы блок знал, кем управлять.
* **`network_id`**: ID кинетической сети, к которой подключен блок.
* **`direction` и `ratio**`: Направление (1 или -1) и передаточное число скорости.
* **`roll`**: не используется, но оставляем для совместимости.
