# API: Сети и Реестр

Для оптимизации работы кинетики, логика хранения и математика вынесены в отдельные модули.

## `KineticNetwork`

Модуль, отвечающий за математику сетей. Хранит глобальную таблицу всех активных механизмов.

* `create(base_rpm)`: Создает новую сеть и возвращает её ID.
* `add_capacity(id, amnt, rpm, ratio, x, y, z)`: Добавляет генерируемую мощность в сеть. Пересчитывает базовый RPM сети, если новый источник быстрее.
* `add_impact(id, amnt, ratio)`: Увеличивает нагрузку на сеть.
* `is_overstressed(id)`: Возвращает `true`, если нагрузка превышает возможности генераторов (`impact * rpm > capacity`).
* `get_rpm(id)`: Возвращает текущую скорость вращения сети. Если сеть перегружена, возвращает `0`.

## `KineticPathfinder`

Отвечает за обход графа блоков в мире (Алгоритм поиска в ширину - BFS).

* `propagate(block0, netid)`: Рекурсивно (через очередь) обходит все соединенные блоки, назначая им `netid`, вычисляя передаточные числа (`ratio`) и направления вращения.
* `destroy_network(block0)`: Уничтожает сеть при удалении блока и заставляет все оставшиеся генераторы пересобрать свои сети.
* `rebuild_network(netid)`: Принудительно пересобирает сеть (полезно при изменении скорости генератора, например, у Hand Crank).

## `KineticRegistry`

Кэширует загруженные кинетические блоки по координатам, чтобы не обращаться к миру постоянно.

* `get_key(x, y, z)`: Генерирует уникальный числовой ключ на основе координат.
* `add(x, y, z, kblock)` / `get(x, y, z)` / `remove(x, y, z)`: Управление кэшем блоков.
